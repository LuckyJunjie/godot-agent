# Godot-Agent End-to-End Game Generation — Gap Analysis & Solutions

**Date:** 2026-05-08  
**Analyst:** Kimi Code CLI  
**Purpose:** Identify architectural gaps that prevent godot-agent from taking a natural-language game requirement and producing a complete, runnable Godot game project.

---

## Scope & Test Artifacts

The snake game artifacts in `tests_output/snake_special/` were **E2E test outputs** from an experimental branch (`origin/master`). They are intentionally incomplete and are **NOT product gaps** — they served their purpose of validating the workflow module's scaffolding ability.

> 🟡 **Snake implementation gaps (incomplete script, bare scene, broken `end` validation) are MARKED AS IGNORED.**  
> 🔴 **This document focuses on DESIGN-LEVEL gaps** — architectural decisions that prevent true end-to-end automation regardless of any single game's implementation completeness.

---

## The Intended Architecture

```
User Requirement (natural language)
    ↓
[GamePlanner]        → LLM decomposes requirement into phases
    ↓
[PhaseExecutor]      → Orchestrates compound tools & godogen skills
    ↓
[Validator]          → LSP + Harness + GDD traceability checks
    ↓
[FixLoop]            → LLM-driven error correction (max N iterations)
    ↓
[Persistent State]   → Resumable game generation progress
    ↓
Complete, runnable Godot game project
```

**Current reality:** The system has two disconnected orchestration layers (agent loop + procedural workflow) with no planning, no compound tools, invisible skills, and no feedback loop.

---

## Design Gap 1: Two Competing Orchestration Layers ⭐ CRITICAL

### Problem

The system has **two mutually exclusive ways to do work** that cannot compose:

1. **Nanobot AgentLoop** (`nanobot/agent/loop.py`) — LLM decides what to do turn-by-turn using available tools. Reactive, no long-term planning. The LLM must micromanage every file edit.
2. **AgentWorkflow** (`godot_agent/workflow/__init__.py`) — Hardcoded procedural pipeline (`parse → GDD → generate → validate`). Bypasses the LLM entirely. Cannot adapt to novel requirements or unexpected errors.

**Evidence:**
- `AgentWorkflow.run()` calls `self.parser.parse()`, `self.parser.to_gdd()`, `self.generator.generate()` in sequence. No LLM involvement after the initial keyword-match parse.
- The nanobot AgentLoop has access to 6 Godot tools but no high-level game generation capability. It would need 50–100 individual tool calls to build a complete game.
- There is no bridge between the two layers. The workflow cannot invoke agent tools, and the agent loop cannot delegate to the workflow.

### Why This Blocks E2E

- The AgentLoop is too low-level for game generation. Asking an LLM to call `gd_scene_edit` 50 times is impractical and error-prone.
- The Workflow is too rigid. It cannot handle requirements like "a multiplayer snake with power-ups and online leaderboards" because the code generator only has hardcoded `if genre == "snake"` branches.
- There is no mechanism for the LLM to **plan** a game, **delegate** execution, **validate** output, and **re-plan** based on validation.

### Detailed Solution

**Replace both layers with a unified meta-agent architecture.**

#### Step 1: Introduce `GamePlanner`

Create `godot_agent/planner/__init__.py`:

```python
"""GamePlanner — LLM-driven decomposition of game requirements into executable phases."""

from dataclasses import dataclass, field
from typing import Optional
import json


@dataclass
class Phase:
    """A single phase of game generation."""
    id: str
    name: str
    description: str
    tool: str  # e.g., "gd_create_component", "gd_asset_generate"
    parameters: dict = field(default_factory=dict)
    prerequisites: list[str] = field(default_factory=list)
    acceptance: list[str] = field(default_factory=list)
    max_retries: int = 3


@dataclass
class GamePlan:
    """A complete plan for generating a game."""
    requirement: str
    genre: str
    phases: list[Phase] = field(default_factory=list)
    current_phase_index: int = 0


class GamePlanner:
    """Uses LLM to decompose a requirement into a GamePlan."""

    PLAN_PROMPT = """You are a game architect. Decompose the following game requirement into a structured implementation plan.

Requirement: {requirement}

Return a JSON object with this structure:
{{
  "genre": "snake|platformer|puzzle|shooter|rpg",
  "phases": [
    {{
      "id": "design_gdd",
      "name": "Design GDD",
      "tool": "gd_gdd_create",
      "parameters": {{"story_id": "001", "title": "..."}},
      "prerequisites": [],
      "acceptance": ["GDD index.md exists", "At least one story defined"]
    }},
    {{
      "id": "scaffold_player",
      "name": "Create Player Scene",
      "tool": "gd_create_component",
      "parameters": {{"component_type": "player", "name": "Player", "behaviors": ["movement", "collision"]}},
      "prerequisites": ["design_gdd"],
      "acceptance": ["scenes/player.tscn exists", "scripts/player.gd exists"]
    }}
  ]
}}

Phases must be ordered by dependency. Use only these tools:
- gd_gdd_create / gd_gdd_read / gd_gdd_validate
- gd_create_component
- gd_asset_generate
- gd_scene_edit
- gd_harness_run
- gd_project_inspect
"""

    def __init__(self, llm_client):
        self.llm = llm_client

    async def plan(self, requirement: str) -> GamePlan:
        """Generate a GamePlan from a natural language requirement."""
        prompt = self.PLAN_PROMPT.format(requirement=requirement)
        response = await self.llm.chat([{"role": "user", "content": prompt}])
        data = json.loads(response)
        phases = [Phase(**p) for p in data["phases"]]
        return GamePlan(
            requirement=requirement,
            genre=data["genre"],
            phases=phases
        )
```

#### Step 2: Introduce `PhaseExecutor`

Create `godot_agent/planner/executor.py`:

```python
"""PhaseExecutor — Runs phases from a GamePlan, handling prerequisites and retries."""

from dataclasses import dataclass
from typing import Optional

from godot_agent.planner import GamePlan, Phase
from godot_agent.bridge.registry import ToolRegistry


@dataclass
class PhaseResult:
    phase_id: str
    success: bool
    output: str
    errors: list[str]
    retry_count: int


class PhaseExecutor:
    """Executes a GamePlan phase by phase, with validation and retry logic."""

    def __init__(self, registry: ToolRegistry, state_path: Optional[str] = None):
        self.registry = registry
        self.state_path = state_path
        self.completed_phases: set[str] = set()
        self.results: list[PhaseResult] = []

    async def execute_plan(self, plan: GamePlan) -> list[PhaseResult]:
        """Execute all phases in dependency order."""
        for phase in plan.phases:
            if not self._prerequisites_met(phase):
                self.results.append(PhaseResult(
                    phase_id=phase.id,
                    success=False,
                    output="",
                    errors=[f"Prerequisites not met: {phase.prerequisites}"],
                    retry_count=0
                ))
                continue

            result = await self._execute_phase_with_retries(phase)
            self.results.append(result)
            if result.success:
                self.completed_phases.add(phase.id)
            else:
                # Stop on first unrecoverable failure
                break

        return self.results

    async def _execute_phase_with_retries(self, phase: Phase) -> PhaseResult:
        """Execute a single phase with retry logic."""
        tool = self.registry.get(phase.tool)
        if not tool:
            return PhaseResult(
                phase_id=phase.id,
                success=False,
                output="",
                errors=[f"Tool {phase.tool} not found in registry"],
                retry_count=0
            )

        for attempt in range(phase.max_retries):
            try:
                output = await tool.execute(**phase.parameters)
                # Run acceptance checks
                errors = self._check_acceptance(phase, output)
                if not errors:
                    return PhaseResult(
                        phase_id=phase.id,
                        success=True,
                        output=output,
                        errors=[],
                        retry_count=attempt
                    )
                # Will retry with error context
            except Exception as e:
                errors = [str(e)]

        return PhaseResult(
            phase_id=phase.id,
            success=False,
            output="",
            errors=errors,
            retry_count=phase.max_retries
        )

    def _prerequisites_met(self, phase: Phase) -> bool:
        return all(p in self.completed_phases for p in phase.prerequisites)

    def _check_acceptance(self, phase: Phase, output: str) -> list[str]:
        """Check if phase output meets acceptance criteria."""
        errors = []
        for criterion in phase.acceptance:
            # Simple heuristic: check if criterion keywords appear in output
            if criterion.lower() not in output.lower():
                errors.append(f"Acceptance criterion not met: {criterion}")
        return errors
```

## Design Gap 2: No High-Level Game Generation Tools ⭐ CRITICAL

### Problem

All 6 Godot bridge tools are **file-level or node-level primitives**. The LLM must micromanage every detail:

| Tool | Granularity | Est. calls for snake game |
|------|-------------|--------------------------|
| `gd_scene_inspect` | Single file | ~5 |
| `gd_scene_edit` | Single node/action | ~20+ |
| `gd_gdd_read` | Single story | ~3 |
| `gd_gdd_validate` | Whole project | ~1/phase |
| `gd_asset_generate` | Single asset | ~5 |
| `gd_harness_run` | Single test | ~3 |

**Total: 50–100 individual tool calls.** LLMs cannot reliably track this many interdependent operations.

### Why This Blocks E2E

- No atomicity: If step 3 fails, steps 1–2 are already committed with no rollback.
- No composition: The LLM cannot say "create a complete player scene" in one call.
- Context window pressure: Tracking 50+ tool results exhausts the LLM's context.

### Detailed Solution

**Add compound tools that operate at the game-component level.**

#### Tool: `gd_create_component`

Create `godot_agent/bridge/tools_compound.py`:

```python
"""Compound tools for high-level game component creation."""

from pathlib import Path
from typing import Any

from nanobot.agent.tools.base import Tool

from godot_agent.scene import SceneDocument
from godot_agent.godogen import GodogenIntegrator
from godot_agent.assets import AssetPipeline, ImageMeta


class CreateComponentTool(Tool):
    """Create a complete game component: scene + script + nodes + assets."""

    @property
    def name(self) -> str:
        return "gd_create_component"

    @property
    def description(self) -> str:
        return (
            "Create a complete game component in one operation. "
            "Generates the GDScript, scene file, required nodes, "
            "and optional sprite asset. "
            "Examples: player character, enemy, collectible, UI screen."
        )

    @property
    def parameters(self) -> dict[str, Any]:
        return {
            "type": "object",
            "properties": {
                "project_root": {"type": "string", "default": "."},
                "component_type": {
                    "type": "string",
                    "enum": ["player", "enemy", "collectible", "obstacle", "ui_screen", "autoload"],
                    "description": "Type of component to create"
                },
                "name": {
                    "type": "string",
                    "description": "Name of the component (e.g., 'Player', 'SlimeEnemy')"
                },
                "behaviors": {
                    "type": "array",
                    "items": {"type": "string", "enum": [
                        "movement", "collision", "animation", "input", "ai",
                        "health", "scoring", "state_machine", "particles"
                    ]},
                    "description": "Behaviors this component needs"
                },
                "extends": {
                    "type": "string",
                    "default": "CharacterBody2D",
                    "description": "Base Godot class to extend"
                },
                "sprite_prompt": {
                    "type": "string",
                    "description": "Optional: prompt for sprite generation"
                },
                "scene_parent": {
                    "type": "string",
                    "default": ".",
                    "description": "Parent node path in scene"
                }
            },
            "required": ["component_type", "name"]
        }

    async def execute(
        self,
        project_root: str = ".",
        component_type: str = "player",
        name: str = "",
        behaviors: list[str] = None,
        extends: str = "CharacterBody2D",
        sprite_prompt: str = "",
        scene_parent: str = "."
    ) -> str:
        root = Path(project_root)
        behaviors = behaviors or []

        # 1. Generate script via godogen skill
        integrator = GodogenIntegrator(project_root)
        integrator.load_all()

        script_content = self._generate_script(
            integrator, component_type, name, behaviors, extends
        )

        script_path = root / "scripts" / f"{name.lower()}.gd"
        script_path.parent.mkdir(parents=True, exist_ok=True)
        script_path.write_text(script_content, encoding="utf-8")

        # 2. Generate scene
        scene_path = root / "scenes" / f"{name.lower()}.tscn"
        scene_path.parent.mkdir(parents=True, exist_ok=True)
        self._generate_scene(scene_path, name, extends, script_path, behaviors, scene_parent)

        # 3. Generate sprite if requested
        asset_path = None
        if sprite_prompt:
            pipeline = AssetPipeline(project_root)
            pipeline.initialize()
            meta = ImageMeta(prompt=sprite_prompt, resolution=(64, 64))
            asset_path = await pipeline.generate_image(meta, name.lower())

        # 4. Return summary
        result = [
            f"✅ Created component '{name}'",
            f"   Script: {script_path}",
            f"   Scene:  {scene_path}",
        ]
        if asset_path:
            result.append(f"   Sprite: {asset_path}")
        return "\n".join(result)

    def _generate_script(self, integrator, component_type, name, behaviors, extends) -> str:
        """Generate GDScript using godogen skills or fallback templates."""
        # Try godogen skill first
        skill_name = f"generate_{component_type}"
        if any(t.name == skill_name for t in integrator.tools):
            return integrator.render_skill(skill_name, {
                "class_name": name,
                "extends": extends,
                "behaviors": behaviors
            })

        # Fallback template
        lines = [
            f"extends {extends}",
            f"class_name {name}",
            "",
            "# Behaviors: " + ", ".join(behaviors),
            "",
        ]
        if "movement" in behaviors:
            lines.extend([
                "@export var speed: float = 200.0",
                "var velocity: Vector2 = Vector2.ZERO",
                "",
                "func _physics_process(delta):",
                "    move_and_slide()",
                "",
            ])
        if "health" in behaviors:
            lines.extend([
                "@export var max_health: int = 100",
                "var health: int = max_health",
                "",
                "func take_damage(amount: int):",
                "    health -= amount",
                "    if health <= 0:",
                "        die()",
                "",
                "func die():",
                "    queue_free()",
                "",
            ])
        if "input" in behaviors:
            lines.extend([
                "func _input(event):",
                "    if event is InputEventKey:",
                "        handle_input(event)",
                "",
                "func handle_input(event: InputEventKey):",
                "    pass",
                "",
            ])
        lines.extend([
            "func _ready():",
            "    pass",
        ])
        return "\n".join(lines)

    def _generate_scene(self, scene_path, name, extends, script_path, behaviors, parent):
        doc = SceneDocument()
        doc.load(str(scene_path)) if scene_path.exists() else None

        # Add ext_resource for script
        doc.add_ext_resource(str(script_path), "Script", f"script_{name.lower()}")

        # Add root node with script
        root_node = doc.add_node(parent, name, extends)
        root_node.set_property("script", f"ExtResource(\"script_{name.lower()}\")")

        # Add child nodes based on behaviors
        if "collision" in behaviors:
            doc.add_node(f"{parent}/{name}", "CollisionShape2D", "CollisionShape2D")
        if "animation" in behaviors:
            doc.add_node(f"{parent}/{name}", "AnimationPlayer", "AnimationPlayer")
        if "particles" in behaviors:
            doc.add_node(f"{parent}/{name}", "Particles", "GPUParticles2D")

        doc.save(str(scene_path))
```

#### Tool: `gd_create_game_from_gdd`

```python
class CreateGameFromGDDTool(Tool):
    """Generate all scenes, scripts, and assets from a GDD story."""

    @property
    def name(self) -> str:
        return "gd_create_game_from_gdd"

    @property
    def description(self) -> str:
        return (
            "Reads a GDD story and generates all required scenes, scripts, "
            "and assets to implement that story. Validates traceability after."
        )

    @property
    def parameters(self) -> dict[str, Any]:
        return {
            "type": "object",
            "properties": {
                "project_root": {"type": "string", "default": "."},
                "story_id": {"type": "string", "description": "Story ID from GDD"}
            },
            "required": ["story_id"]
        }

    async def execute(self, project_root: str = ".", story_id: str = "") -> str:
        from godot_agent.gdd import GDDEngine
        engine = GDDEngine(project_root)
        stories = engine.load_stories()
        story = next((s for s in stories if s.id == story_id), None)
        if not story:
            return f"Story {story_id} not found"

        results = []
        # Generate each referenced scene
        for scene_path in story.scenes:
            scene_name = Path(scene_path).stem
            component_tool = CreateComponentTool()
            result = await component_tool.execute(
                project_root=project_root,
                component_type="player" if "player" in scene_name else "enemy",
                name=scene_name.capitalize()
            )
            results.append(result)

        return f"Generated {len(results)} components for story {story_id}:\n" + "\n".join(results)
```

## Design Gap 3: Godogen Skills Are Invisible to the LLM ⭐ CRITICAL

### Problem

The 5 YAML skill packs in `skills/godogen/` are loaded by `GodogenIntegrator` but:
1. **Not registered as nanobot tools** — LLM cannot discover or invoke them.
2. **Not loaded into the system prompt** — LLM doesn't know they exist.
3. **Not used by AgentWorkflow** — `CodeGenerator` uses hardcoded Python string concatenation.

### Why This Blocks E2E

- Skills represent the agent's "expertise" in code patterns. If the LLM can't use them, it must generate all GDScript from scratch every time.
- A snake game should use `component` for the snake head, `state_machine` for game states, and `ui_screen` for menus. None of this happens.

### Detailed Solution

**Auto-register godogen skills as first-class nanobot tools.**

#### Step 1: Create Dynamic Skill Tool Wrapper

Create `godot_agent/bridge/tools_godogen.py`:

```python
"""Dynamic nanobot Tool wrappers for godogen YAML skills."""

from typing import Any
from nanobot.agent.tools.base import Tool
from godot_agent.godogen import GodogenIntegrator, ToolSpec


class GodogenSkillTool(Tool):
    """A nanobot Tool backed by a godogen YAML skill."""

    def __init__(self, spec: ToolSpec, integrator: GodogenIntegrator):
        self._spec = spec
        self._integrator = integrator

    @property
    def name(self) -> str:
        return f"godogen_{self._spec.name}"

    @property
    def description(self) -> str:
        return (
            f"Godogen skill: {self._spec.name}\n"
            f"{self._spec.description}\n"
            f"Renders a Jinja2 template into GDScript, scene, or config code."
        )

    @property
    def parameters(self) -> dict[str, Any]:
        schema = self._spec.input_schema or {"type": "object", "properties": {}}
        # Ensure all godogen parameters are required by default
        if "required" not in schema:
            schema["required"] = list(schema.get("properties", {}).keys())
        return schema

    async def execute(self, **kwargs) -> str:
        rendered = self._integrator.render_skill(self._spec.name, kwargs)
        if not rendered:
            return f"Error: skill '{self._spec.name}' returned empty output"
        return rendered
```

## Design Gap 4: No Script-Aware Editing Tool 🟠 HIGH

### Problem

Nanobot provides generic `write_file` and `edit_file` tools. These are text-based and GDScript-naïve. The LSP client exists but is **not exposed as a tool**.

### Why This Blocks E2E

- The LLM can write GDScript with syntax errors (wrong built-in names, missing type hints).
- Without LSP feedback, errors are only caught at test time — too late in the loop.
- The LLM cannot use "go to definition" to understand existing code before modifying it.

### Detailed Solution

**Expose LSP capabilities as nanobot tools.**

Create `godot_agent/bridge/tools_lsp.py`:

```python
"""LSP-based tools for GDScript-aware editing."""

from typing import Any
from nanobot.agent.tools.base import Tool
from godot_agent.lsp import GDScriptLSPClient


class GDScriptLintTool(Tool):
    """Run GDScript linting via Godot LSP and return diagnostics."""

    @property
    def name(self) -> str:
        return "gd_script_lint"

    @property
    def description(self) -> str:
        return (
            "Check a GDScript file for syntax errors, type mismatches, "
            "and warnings using the Godot LSP server. "

## Design Gap 5: No Validation-as-Feedback Loop 🟠 HIGH

### Problem

The harness can run tests (`gd_harness_run`), but:
1. **No automatic retry.** If a test fails, the agent loop does not automatically feed the error back to the LLM.
2. **No fix tool.** There's no `gd_fix_script(path, error)` tool.
3. **Workflow's FixLoop is procedural, not LLM-driven.** `test_fix.py` does regex replacements instead of asking the LLM to reason about errors.

### Why This Blocks E2E

- Generated games will have bugs. The system must detect them (via tests/LSP) and fix them (via LLM reasoning).
- Currently, validation is terminal. There's no "validation failed → re-generate → re-validate" loop.

### Detailed Solution

**Implement a TDD loop within the agent.**

Create `godot_agent/planner/fix_loop.py`:

```python
"""FixLoop — LLM-driven error correction with validation feedback."""

from dataclasses import dataclass
from typing import Optional
import json


@dataclass
class FixIteration:
    iteration: int
    errors: list[str]
    fix_applied: str
    validation_result: str


class FixLoop:
    """Iteratively fixes code based on validation errors."""

    MAX_ITERATIONS = 5

    FIX_PROMPT = """You are a Godot game developer fixing errors in generated code.

File: {file_path}
Current code:
```gdscript
{code}
```

Validation errors:
{errors}

Please provide the COMPLETE fixed version of the file.
Return ONLY the corrected code inside a ```gdscript block.
Do not explain the changes.
"""

    def __init__(self, llm_client, tool_registry):
        self.llm = llm_client
        self.registry = tool_registry

    async def fix(
        self,
        file_path: str,
        validation_errors: list[str],
        max_iterations: int = MAX_ITERATIONS
    ) -> tuple[bool, list[FixIteration]]:
        """Attempt to fix a file based on validation errors.

        Returns (success, iterations_log).
        """
        from pathlib import Path
        code = Path(file_path).read_text(encoding="utf-8")
        iterations = []

        for i in range(max_iterations):
            # Ask LLM to fix
            prompt = self.FIX_PROMPT.format(
                file_path=file_path,
                code=code,
                errors="\n".join(f"- {e}" for e in validation_errors)
            )
            response = await self.llm.chat([{"role": "user", "content": prompt}])
            fixed_code = self._extract_code(response)

            if not fixed_code:
                iterations.append(FixIteration(
                    iteration=i,
                    errors=validation_errors,
                    fix_applied="[LLM did not return code]",
                    validation_result="FAILED"
                ))
                continue

            # Apply fix
            Path(file_path).write_text(fixed_code, encoding="utf-8")

            # Re-validate
            lint_tool = self.registry.get("gd_script_lint")
            harness_tool = self.registry.get("gd_harness_run")

            new_errors = []
            if lint_tool:
                lint_result = await lint_tool.execute(path=file_path)
                if "❌" in lint_result:
                    new_errors.append(lint_result)
            if harness_tool and file_path.endswith(".gd"):
                test_result = await harness_tool.run_unit(file_path)
                if not test_result.passed:
                    new_errors.append(test_result.message)

            iterations.append(FixIteration(
                iteration=i,
                errors=validation_errors,
                fix_applied=fixed_code[:200] + "...",
                validation_result="PASSED" if not new_errors else "FAILED"
            ))

            if not new_errors:
                return True, iterations

            validation_errors = new_errors
            code = fixed_code

        return False, iterations

    def _extract_code(self, response: str) -> Optional[str]:
        """Extract code from markdown code block."""
        if "```gdscript" in response:
            start = response.find("```gdscript") + len("```gdscript")
            end = response.find("```", start)
            return response[start:end].strip()
        if "```" in response:
            start = response.find("```") + 3
            end = response.find("```", start)
            return response[start:end].strip()
        return response.strip()
```

## Design Gap 6: No Multi-File Atomic Transactions 🟠 HIGH

### Problem

Each tool call is independent and immediately commits to disk. There's no "transaction" spanning multiple files.

### Why This Blocks E2E

- Game generation is inherently multi-file. A scene references a script, which references an asset.
- If step 3 fails, steps 1–2 are already committed. No rollback.
- Partial failures leave the project in an inconsistent state.

### Detailed Solution

**Implement a staging / preview mechanism.**

Create `godot_agent/staging/__init__.py`:

```python
"""Staging area for atomic multi-file game generation operations."""

import shutil
import tempfile
from dataclasses import dataclass, field
from pathlib import Path
from typing import Optional


@dataclass
class StagedChange:
    """A single file change in the staging area."""
    target_path: str
    content: str
    operation: str = "write"  # write | delete | mkdir


class StagingArea:
    """Accumulates changes in a temp directory before atomic application."""

    def __init__(self, project_root: str):
        self.project_root = Path(project_root)
        self.temp_dir = Path(tempfile.mkdtemp(prefix="godot_agent_staging_"))
        self.changes: list[StagedChange] = []
        self._original_backup: Optional[Path] = None

    def write_file(self, rel_path: str, content: str) -> None:
        """Stage a file write."""
        target = self.temp_dir / rel_path
        target.parent.mkdir(parents=True, exist_ok=True)
        target.write_text(content, encoding="utf-8")
        self.changes.append(StagedChange(rel_path, content, "write"))

    def delete_file(self, rel_path: str) -> None:
        """Stage a file deletion."""
        self.changes.append(StagedChange(rel_path, "", "delete"))

    def preview(self) -> str:
        """Return a human-readable preview of all staged changes."""
        lines = ["=== Staged Changes ==="]
        for change in self.changes:
            if change.operation == "write":
                lines.append(f"  WRITE {change.target_path} ({len(change.content)} chars)")
            elif change.operation == "delete":
                lines.append(f"  DELETE {change.target_path}")
        return "\n".join(lines)

    def apply(self) -> None:
        """Atomically apply all staged changes to the project."""
        # Create backup
        self._original_backup = Path(tempfile.mkdtemp(prefix="godot_agent_backup_"))
        for change in self.changes:
            src = self.project_root / change.target_path
            if src.exists():
                dst = self._original_backup / change.target_path
                dst.parent.mkdir(parents=True, exist_ok=True)
                if src.is_file():
                    shutil.copy2(src, dst)
                else:
                    shutil.copytree(src, dst)

        # Apply changes
        for change in self.changes:
            target = self.project_root / change.target_path
            if change.operation == "write":
                target.parent.mkdir(parents=True, exist_ok=True)
                target.write_text(change.content, encoding="utf-8")
            elif change.operation == "delete":
                if target.exists():
                    target.unlink()

    def rollback(self) -> None:
        """Restore project from backup."""
        if not self._original_backup:
            return
        for change in self.changes:
            target = self.project_root / change.target_path
            backup = self._original_backup / change.target_path
            if backup.exists():
                target.parent.mkdir(parents=True, exist_ok=True)
                shutil.copy2(backup, target)
            elif target.exists():
                target.unlink()

    def cleanup(self) -> None:
        """Remove temp directories."""
        shutil.rmtree(self.temp_dir, ignore_errors=True)
        if self._original_backup:
            shutil.rmtree(self._original_backup, ignore_errors=True)
```

Create `godot_agent/state/__init__.py`:

```python
"""Persistent state for game generation projects."""

import json
from dataclasses import dataclass, field, asdict
from pathlib import Path
from typing import Optional


@dataclass
class GameProjectState:
    """Tracks the progress of a game generation project."""

    requirement: str
    project_root: str
    genre: str = ""
    current_phase: str = "idle"  # idle | designing | scaffolding | implementing | testing | polishing | complete
    completed_phases: list[str] = field(default_factory=list)
    pending_phases: list[str] = field(default_factory=list)
    known_issues: list[dict] = field(default_factory=list)
    generated_files: list[str] = field(default_factory=list)
    gdd_path: str = ""
    last_error: str = ""
    metadata: dict = field(default_factory=dict)

    @classmethod
    def load(cls, project_root: str) -> Optional["GameProjectState"]:
        """Load state from disk."""
        path = Path(project_root) / ".godot-agent" / "state.json"
        if not path.exists():
            return None
        data = json.loads(path.read_text(encoding="utf-8"))
        return cls(**data)

    def save(self) -> None:
        """Save state to disk."""
        path = Path(self.project_root) / ".godot-agent" / "state.json"
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_text(json.dumps(asdict(self), indent=2), encoding="utf-8")

    def mark_phase_complete(self, phase: str) -> None:
        if phase in self.pending_phases:
            self.pending_phases.remove(phase)
        if phase not in self.completed_phases:
            self.completed_phases.append(phase)
        self.save()

    def add_issue(self, phase: str, description: str, severity: str = "warning") -> None:
        self.known_issues.append({
            "phase": phase,
            "description": description,
            "severity": severity,
            "resolved": False
        })
        self.save()

    def add_generated_file(self, file_path: str) -> None:
        if file_path not in self.generated_files:
            self.generated_files.append(file_path)
            self.save()

    def get_status(self) -> str:
        lines = [
            f"Project: {self.project_root}",
            f"Phase: {self.current_phase}",
            f"Completed: {len(self.completed_phases)} / {len(self.completed_phases) + len(self.pending_phases)}",
            f"Files: {len(self.generated_files)}",
            f"Issues: {len([i for i in self.known_issues if not i['resolved']])} open",
        ]
        return "\n".join(lines)
```

### Problem

### Why This Blocks E2E

- Even with perfect tools, an LLM needs guidance on HOW to use them for a complex goal.

### Detailed Solution

**Create a Game Development Playbook prompt and inject it into the agent context.**

Replace `godot_agent/templates/agent/identity.md` with:

```markdown
# Godot Agent — Game Development Playbook

You are Godot Agent, an autonomous game developer specializing in Godot 4.x.
Your mission: take a natural language game requirement and produce a complete, runnable Godot project.

## Core Philosophy

1. **GDD-first** — Always create or read the Game Design Document before writing code.
2. **Component-driven** — Build games from reusable components, not monolithic scripts.
3. **Test-as-you-go** — Validate every component before building the next one.
4. **Skills-first** — Use godogen skills for code patterns before writing custom code.

## Workflow (MANDATORY)

When asked to create a game, follow these phases IN ORDER:

### Phase 1: Design (1–2 turns)
1. Call `gd_gdd_read` to check for existing GDD.
2. If no GDD exists, create one:
   - Use `gd_gdd_create` to initialize the GDD structure.
   - Create stories for each major feature.
   - Define acceptance criteria for each story.
3. Validate with `gd_gdd_validate`.

### Phase 2: Scaffold (1–2 turns)
For each component identified in the GDD:
1. **Prefer godogen skills**:
   - Player movement → `godogen_top_down_movement` or `godogen_component`
   - Game states → `godogen_state_machine`
   - UI → `godogen_ui_screen`
   - Input → `godogen_input_map`
2. If no skill matches, use `gd_create_component` to generate scene + script.
3. Use `gd_scene_edit` only for fine-tuning after component creation.

### Phase 3: Implement (3–5 turns per mechanic)
1. Implement core mechanics in scripts using `write_file` or `edit_file`.
2. After EACH script change, run `gd_script_lint` for syntax checking.
3. After EACH scene change, run `gd_scene_inspect` to verify structure.

### Phase 4: Assets (1–2 turns)
1. For each required sprite/sound, call `gd_asset_generate`.
2. Reference generated assets in scenes using `gd_scene_edit`.

### Phase 5: Validate (1 turn)
1. Run `gd_harness_run` for all test scripts.
2. Run `gd_gdd_validate` for traceability.
3. Run `gd_project_inspect` for warnings.
4. If any step fails, enter the FixLoop:
   - Run `gd_script_lint` to identify errors.
   - Fix errors with `edit_file` or by re-invoking the appropriate godogen skill.
   - Re-run validation.
   - Repeat up to 3 times.

### Phase 6: Export (optional)
1. Call `gd_export_game` to create a runnable build.

## Tool Selection Rules

| Task | Preferred Tool | Fallback |
|------|---------------|----------|
| Create new scene + script | `gd_create_component` | `write_file` + `gd_scene_edit` |
| Add code pattern | `godogen_<skill>` | `write_file` with template |
| Edit existing script | `edit_file` | `write_file` (full overwrite) |
| Check syntax | `gd_script_lint` | `gd_harness_run` |
| Generate sprite | `gd_asset_generate` (image) | Placeholder PNG |
| Run tests | `gd_harness_run` | Manual review |

## Anti-Patterns (NEVER DO)

- ❌ Write a 500-line monolithic script. Use components.
- ❌ Generate assets before designing the GDD.
- ❌ Skip validation after making changes.
- ❌ Use `gd_scene_edit` for bulk node creation. Use `gd_create_component`.
- ❌ Ignore linter warnings. Fix them immediately.
```

## Design Gap 10: No Game-Specific Reasoning Strategy 🟢 LOW

The `GamePlanner` enforces phase ordering via prerequisites. The Playbook prompt teaches the LLM the correct sequence. Together they provide the game-specific reasoning strategy.

**Additional enhancement:** Add dependency-aware scheduling to `PhaseExecutor`:

```python
def _ topological_sort(phases: list[Phase]) -> list[Phase]:
    """Sort phases so prerequisites come before dependents."""
    graph = {p.id: p for p in phases}
    in_degree = {p.id: len(p.prerequisites) for p in phases}
    queue = [p for p in phases if in_degree[p.id] == 0]
    sorted_phases = []
    while queue:
        phase = queue.pop(0)
        sorted_phases.append(phase)
        for other in phases:
            if phase.id in other.prerequisites:
                in_degree[other.id] -= 1
                if in_degree[other.id] == 0:
                    queue.append(other)
    return sorted_phases
```
